## 摘要
近年来，多模态大语言模型（MLLMs）在“用图像思考”（TwI）范式下，通过迭代执行各种视觉工具操作，在细粒度感知任务上取得了显著进展。然而，这一范式严重依赖频繁的外部工具调用和重复的图像重新编码，导致了巨大的计算开销和推理延迟。为了解决这些问题，我们提出了超越视觉（BEE），一种以自我调节能力为核心的新型隐式视觉工具范式。BEE将视觉工具调用行为直接纳入训练目标，并鼓励模型发展出一种自我调节的调用机制。这种设计使模型能够自适应地平衡内部知识和隐式工具，在显著降低推理延迟的同时避免冗余工具使用。具体来说，BEE包含两个训练阶段：（1）形式化思维链（CoT）监督微调（SFT）。我们构建了带有结构化工具槽和混合调用状态的CoT轨迹。该阶段激活了模型的隐式工具表示和自适应切换能力。（2）自我调节的奖励驱动对齐。为了解决因模糊认知边界导致的冗余工具使用问题，我们首先引入净工具增益（NTG）指标来量化这一现象。基于此观察，我们进一步提出了一种自我调节的奖励机制。该机制惩罚无效的工具依赖，并鼓励模型进行知识路由，确保仅在模型内部知识不足时才调用隐式工具。BEE在细粒度视觉感知方面达到了最先进的性能，同时在通用推理任务中保持竞争力，并在推理效率上取得了显著提升。

## Abstract
Recent multimodal large language models (MLLMs) have made remarkable progress on fine-grained perception tasks under the "Thinking with Images" (TwI) paradigm by iteratively performing various visual tool operations. However, this paradigm relies heavily on frequent external tool calls and repeated image re-encoding, which leads to substantial computational overhead and inference latency. To address these issues, we propose Beyond the Eye (BEE), a novel implicit visual tool paradigm centered on self-regulated capability. BEE directly incorporates visual tool invocation behaviors into the training objective and encourages the model to develop a self-regulated invocation mechanism. This design enables the model to adaptively balance internal knowledge and implicit tools, avoiding redundant tool usage while substantially reducing inference latency. Specifically, BEE involves a two-stage training process: (1) Formalized Chain-of-Thought (CoT) Supervised Fine-tuning (SFT). We construct CoT trajectories with structured tool slots and mixed invocation states. This stage activates the model's implicit tool representations and adaptive switching capability. (2) Self-regulated Reward-Driven Alignment. To address redundant tool usage caused by ambiguous cognitive boundaries, we first introduce the Net Tool Gain (NTG) metric to quantify this phenomenon. Based on this observation, we further propose a self-regulated reward mechanism. This mechanism penalizes ineffective tool dependency and encourages the model to perform knowledge routing, ensuring that implicit tools are invoked only when the model's internal knowledge is insufficient. BEE achieves state-of-the-art performance in fine-grained visual perception while remaining competitive in general reasoning tasks and achieving substantial gains in inference efficiency.

## 1. 核心问题与研究动机

当前多模态大语言模型（MLLMs）在“用图像思考”（Thinking with Images, TwI）范式下，通过频繁调用外部视觉工具（如缩放、旋转、裁剪）并进行重复图像编码来实现细粒度感知。这带来了两大问题：
- **高计算开销与推理延迟**：外部工具调用产生 I/O 操作，重复图像编码增加额外成本。
- **缺乏能力边界意识**：模型无法自适应判断何时依赖内部参数知识、何时调用外部工具，导致大量冗余工具使用。

为此，论文提出 **Beyond the Eye (BEE)**，一种通过**自调节**能力实现隐式工具调用的新范式，旨在平衡推理准确性与效率。

## 2. 方法论：核心思想与关键技术细节

BEE 将视觉工具调用行为内化到训练目标中，并引入自调节机制，使模型自适应选择是否使用隐式工具。

### 2.1 两阶段训练流程
- **阶段一：形式化 CoT 监督微调 (Formalized CoT SFT)**  
  构建包含结构化工具槽（`<tool_pad>`）和混合调用状态（有/无工具调用）的 CoT 轨迹，激活模型的隐式工具表示与自适应切换能力。  
  数据集 BEE-SFT-320K 来源于 Refocus、Thyme、mini-o3、Visual-CoT 等，涵盖六种视觉操作（缩放、裁剪、旋转等），并混合了不需工具的样本。

- **阶段二：自调节奖励驱动对齐 (Self-regulated Reward-Driven Alignment)**  
  引入 **Net Tool Gain (NTG)** 指标量化工具调用带来的实际边际收益（式1）。  
  提出 **MC-Reward** 自调节奖励机制（式2-10），基于组内平均准确率 \( p \) 动态判断任务难度：
  - 当 \( p < \tau \)（困难任务）时，鼓励调用工具；
  - 当 \( p \geq \tau \)（简单任务）时，惩罚不必要的工具调用。
  MC-Reward 通过连续余弦函数实现平滑过渡，避免离散奖励带来的训练不稳定。

### 2.2 隐式工具表示
BEE 不依赖外部 API 调用，而是通过预训练的语义标记器（Semantic Tool Tokenizer）将工具操作结果编码为隐式离散 token，在自回归生成中被模型内部使用，实现单次前向传播内的工具增强推理。

### 2.3 算法流程
**Algorithm 1** 简述：
1. 在形式化 CoT SFT 数据集上训练 πθ 得到 πθ,SFT。
2. 在 RL 阶段，对每组查询进行多次 rollout，使用自调节奖励（含正确性、格式、一致性、MC-Reward）计算 reward，以 DAPO（式3）更新策略。

## 6. 主要结论与发现

- BEE 在多个细粒度感知任务上达到 SOTA，同时保持通用推理竞争力。
- 自调节奖励（MC-Reward）有效抑制冗余工具调用，使 NTG 从 6.2% 提升至 66.0%（Table 7），工具调用率趋于合理（简单问题 < 0.5%，困难问题约 41%）。
- 推理效率显著优于显式工具方法：8K 分辨率下加速 86.2%，成本降低至 DeepEyes 的 1/9 (Table 4)。
- 感知与推理解耦（adapter 结构）可进一步提升性能（Table 10）。

## 7. 优点与亮点

- **端到端隐式工具范式**：消除外部 API 调用与重复图像编码，降低延迟与级联错误。
- **自调节能力边界意识**：通过 MC-Reward 动态调整工具使用，实现自适应计算分配。
- **训练资源需求低**：在单节点 8 GPU 上即可完成训练，远低于同类方法。
- **强性能与高效率并存**：在不牺牲精度前提下大幅提升吞吐量。
- **广泛的消融与诊断分析**：深入剖析隐式 token 作用、感知推理分离等设计选择，增强可信度。

## 8. 不足与局限

- **复杂逻辑推理能力有限**：在 VisuLogic 等基准上表现一般，原因是训练数据中缺乏复杂推理样例。  
- **多相似目标遗漏检测**：在自动驾驶等密集场景中，可能遗漏部分车辆/目标，源于训练数据缺乏此类多物体感知案例。
- **领域泛化待加强**：某些 OOD 任务（如特定医学图像）未充分验证，可能受限于数据分布。
- **隐式表示可解释性不足**：虽然设计了 token 理解辅助训练，但隐式特征的具体语义仍不够透明。
- **阈值 τ 设置需人工调整**：实验中 τ=2/3 表现最佳，但理论上的最优选择未深入探讨。
