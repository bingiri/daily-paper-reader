## 摘要
推理时序定位要求模型生成的答案本身包含支持其推理的时间区间，因此必须在单一响应中同时完成高级推理与精确的时间定位。为应对这一挑战，我们提出了首个以事件为中心的视频思维链框架，名为EventCoT。EventCoT首先对输入视频进行以事件为中心的标记化，将其转换为紧凑的事件标记，从而高效识别与问题相关的事件；随后在识别的事件中进行推理以生成答案，并通过嵌入匹配将占位符标记与视觉嵌入对齐，从而实现时间区间的定位。在推理时序定位任务ActivityNet-RTL上，EventCoT取得了最先进的结果，同时使用的视觉标记数量远少于先前工作。为验证其通用性能，我们在接地视频问答基准ReXTime上进一步评估EventCoT，该方法在零样本场景下取得了强劲的结果。

## Abstract
Reasoning temporal localization (RTL) requires a model to generate an answer that itself contains the time interval supporting it, so high-level reasoning and precise temporal grounding must be produced jointly in a single response. To tackle this challenging task, we propose the first event-centric video chain-of-thought framework, dubbed EventCoT. EventCoT first performs event-centric tokenization of the input video to convert it into compact event tokens, enabling efficient identification of question-relevant events. It then reasons within the identified events to generate the answer, grounding the time interval via embedding matching that aligns placeholder tokens with visual embeddings. EventCoT achieves state-of-the-art results on ActivityNet-RTL for reasoning temporal localization while using substantially fewer visual tokens than previous work. To verify its general performance, we further evaluate EventCoT on the grounded video question answering benchmark ReXTime, where it attains strong zero-shot results.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：推理时序定位（Reasoning Temporal Localization, RTL）要求模型生成的答案**本身就包含支持该答案的时间区间**（起始与结束时间戳）。这与传统视频问答不同——传统任务中时间定位仅是辅助输出，而RTL要求推理和定位在同一响应中联合生成，极具挑战性。
- **背景与动机**：现有视频大语言模型（Video-LLMs）虽然能处理长视频和高层次推理，但缺乏精确的时间推理能力。早期工作（如LITA）通过均匀采样帧并使用离散时间令牌预测时间区间，存在两大缺陷：①查询无关的均匀采样带来大量冗余噪声，尤其在长视频中仅小部分相关；②时间区间预测与答案生成纠缠在一起，既降低定位精度又拖慢推理。后续的智能体系统（如VideoMind、CAViAR）通过多轮检索缓解了第一个问题，但计算成本高，且未解决纠缠问题。因此，需要一种能**解耦推理与定位、同时保持高效**的框架。

### 2. 论文提出的方法论：核心思想、关键技术细节、流程说明

- **核心思想**：以事件为中心的链式思维（Event-Centric Chain-of-Thought），即**先粗粒度选择与问题相关的事件，再在选定事件内进行细粒度推理并生成带时间戳的答案**，从而分离推理与定位。
- **关键技术细节**：
  1. **事件标记化（Event-Centric Tokenization）**：
     - 均匀采样T=100帧，用CLIP提取帧级特征。
     - 采用改进的密度峰值聚类（DPC-KNN）检测事件边界：结合帧间特征距离和时序邻近性（α=0.3），并应用幂缩放距离因子（p=2），输出边界分数，选取前N-1个边界划分出N=16个事件。
     - 对每个事件，先通过均值池化得到事件点特征，再通过事件上下文注意力（Event Context Attention）聚合边界扩展区域（β=3帧）的帧特征，生成紧凑的事件令牌。
  2. **两步CoT推理**：
     - **Step 1：选择与问题相关的事件**。引入占位符令牌`<segment_key>`，其隐藏嵌入与每个事件令牌的嵌入计算余弦相似度（经sigmoid归一化），选择得分高于阈值γ·max的事件。
     - **Step 2：在选定事件内进行细粒度推理**。将选定事件的帧特征（并随机混入30%的未选定事件帧以打破捷径）输入LLM，同时引入`<start>`和`<end>`占位符令牌。在生成答案时，这些占位符的隐藏嵌入与帧级视觉嵌入进行匹配（softmax相似度），选出最相似帧对应的时刻，从而得到时间戳。
  3. **训练目标**：
     - 语言建模损失（标准交叉熵）。
     - 事件选择损失（二分类交叉熵，根据事件与真实时间区间是否有重叠定义标签）。
     - 时间定位损失：高斯平滑标签的交叉熵 + DIoU损失，共同优化`<start>`和`<end>`的匹配概率。

### 7. 优点

- **创新性**：首次将事件级链式思维引入推理时序定位，利用事件作为中间推理单元，自然分离粗选与细粒度推理。
- **高效性**：视觉令牌使用量极少（16个事件令牌 + 约39个帧级令牌），大幅降低LLM计算开销（推理时间对比图显示约为LITA的1/10~1/5）。

- **泛化性**：在零样本ReXTime上表现强劲，证明方法不局限于训练分布。
- **实验设计全面**：消融涵盖方法各部分和超参数，且提供特征判别性分析和推理时间分析，验证设计合理性。

### 8. 不足与局限

- **固定事件数与帧数**：当前固定N=16、T=100，不适应不同长度视频，长视频中可能合并或碎片化事件。论文也承认这是未来方向（自适应事件数、可变帧数）。
- **占位符令牌双重角色**：`<segment_key>`等令牌同时受语言建模损失和匹配损失优化，单一嵌入需兼顾生成与匹配，可能限制性能。论文建议引入潜在CoT（Latent CoT）来分离角色。
- **缺少空间细节**：目前使用平均池化的帧级特征，无法处理需要精细空间定位的查询（如“人在图像左边做什么”）。未来可结合空间特征图。
- **基准局限性**：ActivityNet-RTL仅229个样本，规模较小；ReXTime零样本实验未在测试集上微调，但训练数据包含ActivityNet-Captions等，存在数据重叠风险（但论文采用零样本设置评估，可接受）。
- **未提及偏差分析**：未讨论训练数据中时间标注的偏差或模型对特定时长/事件类型的偏好。
