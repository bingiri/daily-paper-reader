## 摘要
长音频-视频推理对于全模态大语言模型而言是困难的，因为决定性证据往往稀疏、跨模态，且以均匀高保真输入保存成本过高。我们提出OmniReasoner，这是一个用于长音频-视频推理的工具使用后训练框架：全模态大语言模型通过监督微调和强化学习，学习在回答前决定是否以及在哪里调用放大工具。OmniReasoner首先构建整个流的低成本全局预览，然后在需要时调用放大工具并请求一个时间间隔，以便在回答前进行更高保真度的视觉和音频检查。由于模型在调用前后观察到不同的采样粒度——稀疏的全局预览和密集的局部片段——我们引入了TimeAnchor，它使工具的时间参数保持有效，并在这些粒度之间保持往返一致性，而不是与特定采样率的帧索引绑定。为了使这种工具使用行为可训练而无需昂贵的手动间隔标注，我们构建了一个时间增强数据引擎，通过视频编辑和合成来生成工具使用的后训练轨迹。跨全模态和视频基准的实验表明，OmniReasoner提高了答案准确性和时间定位能力，同时将高保真计算集中在信息丰富的区域。代码可在https://github.com/RockyChen0205/OmniReasoner获取。

## Abstract
Long audio-video reasoning is difficult for omnimodal LLMs because the decisive evidence is often sparse, cross-modal, and too expensive to preserve with uniformly high-fidelity inputs. We introduce OmniReasoner, a tool-use post-training framework for Thinking with Long Audio-Video: omni-modal LLMs learn, via supervised fine-tuning and reinforcement learning, to decide whether and where to call a zoom-in tool before answering. OmniReasoner first builds a low-cost global preview of the full stream and then, when needed, calls the zoom-in tool with a requested temporal interval for higher-fidelity visual and audio inspection before answering. Because the model observes different sampling granularities before and after this call -- a sparse global preview and a denser local clip -- we introduce TimeAnchor, which keeps the tool's temporal argument valid and round-trip-consistent across these granularities, rather than tied to frame indices from a particular sampling rate. To make this tool-use behavior trainable without expensive manual interval annotation, we build a Temporal Augmented Data Engine that synthesizes tool-use post-training trajectories by video editing and composition. Experiments across omnimodal and video benchmarks show that OmniReasoner improves both answer accuracy and temporal grounding while concentrating high-fidelity computation on informative regions. Code is available at https://github.com/RockyChen0205/OmniReasoner.

## 1. 论文的核心问题与整体含义

- **研究动机与背景**：长音频-视频推理中，关键证据往往稀疏、跨模态（如短暂的视觉动作、语音片段、背景声音或音视频时间巧合），且以均匀高保真方式处理整个流会消耗大量上下文，成本过高。现有方法多聚焦于视觉证据，缺乏对音频-视觉联合自适应感知的机制。
- **核心问题**：如何让全模态大语言模型学会在长音频-视频流中自适应地决定 **何时** 需要额外的证据，以及 **何处**（时间位置）投入高保真计算，从而在保持全局覆盖的同时集中资源到关键片段。

## 2. 方法论

- **核心思想**：提出 OmniReasoner，一个工具使用后训练框架。模型通过监督微调（SFT）和强化学习（RL）学习在回答前调用一个 **zoom-in 工具**，获取更高保真度的局部音频-视频片段。
- **关键技术细节**：
  - **推理流程**：
    1. **全局观察**：以低帧率/低分辨率消费整个流的低成本全局预览（音频全部处理）。
    2. **决策**：基于全局观察和问题，模型决定直接回答（`answer()`）或请求一个时间区间（`zoom([s, e])`）进行局部高保真检查。
    3. **局部观察**：若调用 zoom，返回该区间的更稠密采样音频-视频片段。
    4. **最终回答**：结合全局和局部证据生成答案。
  - **TimeAnchor**：为解决 zoom 调用前后采样粒度不一致（稀疏全局 vs. 稠密局部）导致的时间参数错位问题，在每个2秒块前加入纯文本的绝对时间标记 `<t seconds>`（如 `<0 seconds>`），使模型发出的时间区间以绝对秒为单位，跨粒度保持一致。
  - **数据引擎（Temporal Augmented Data Engine）**：通过视频编辑和合成自动生成工具使用轨迹，包括：
    - **多片段组合**：将多个语义无关片段拼接成长视频，每个问题对应一个片段，其偏移即为证据区间。
    - **异常插入**：在长视频中替换音频、视频或两者，模型需检测异常类型与位置。
    - **轨迹合成**：使用 Gemini 在线（无参考）或事后（有参考但防泄露）方式生成轨迹，并经过多重过滤（结构、泄露、答案、LLM 评判等）。
  - **后训练**：先 SFT 学习工具使用格式与基本策略（25,839 例），再通过 GRPO（Group Relative Policy Optimization）优化决策质量（2,731 例），奖励仅基于答案正确性与格式合法性（无单独定位奖励）。

## 6. 论文的主要结论与发现

- OmniReasoner 在音视频与通用视频基准上显著优于基座模型 Qwen2.5-Omni-7B，尤其在依赖稀疏证据定位的长视频场景中增益最大（如 OmniVideoBench 提升 +5.5，VideoHolmes 提升 +15.6）。
- 随视频时长增加，工具调用率上升（0-1 分钟段 33.3% → 10-30 分钟段 84.8%），且工具使用模式下的准确率通常高于直接回答模式，说明模型自适应地将高保真计算分配给长视频中的关键片段。
- TimeAnchor 有效解决了跨采样粒度的时间对齐问题，在 Charades-STA 上 IoU@0.3 从 58.8% 提升至 64.9%，并间接改善问答准确率。
- 注意力可视化表明模型确实将注意力集中在 zoom 返回的片段上，而非仅装饰性使用。

## 7. 优点

- **创新性**：将工具使用与后训练（SFT + RL）结合，使模型自主学习何时、何处调用聚焦工具，突破了传统均匀处理或固定证据选择的方式。
- **TimeAnchor 简单有效**：用文本标记实现跨粒度时间一致性，无需修改架构、新增特殊 token 或单独定位损失，设计优雅且可迁移。
- **数据引擎自动化**：通过视频编辑构造有标注证据区间的任务，合成工具使用轨迹，大幅降低人工标注成本，并提供了可扩展的数据生成方案。
- **实验扎实**：多维度消融和子组分析验证了各组件的必要性，注意力可视化增强了模型可解释性。
- **代码开源**：提供完整代码，便于复现和后续研究。

## 8. 不足与局限

- **工具范围有限**：仅实现了一个聚焦感知工具（zoom-in），未探索检索外部知识、代码执行、结构化查询等互补工具，限制了处理需要综合事实与推理的任务的能力。
- **基础模型与基础设施限制**：仅在 Qwen2.5-Omni-7B（32K 上下文窗口）上实验，未在更大模型（如 Qwen-Omni-3）上验证；多轮工具使用时 RL 框架不成熟，需要自行搭建，且缺乏对多步交互的原生支持。
- **冷启动依赖**：直接 RL 训练效果极差（模型倾向不调用工具或输出格式错误），必须依赖 SFT 暖启动，这表明当前的 RL 算法对工具使用行为的学习仍不够鲁棒。
- **潜在偏差与幻觉风险**：合成数据可能存在质量噪音；模型可能产生错误的时间定位或幻觉证据，在高风险场景下需要人工核查。
- **伦理考虑**：使用的公开数据集可能包含可识别人物、冒犯性内容等，在公开发布前需进行许可和隐私审查。
